# HTML

# POO
Paradigma onde o código é organizado em objetos que combinam dados (atributos) e comportamentos (métodos). Os pilares são encapsulamento, herança, polimorfismo e abstração. Exemplos de linguagens: Java, Python, C#.

# POO VS PE
PE é mais simples e direto — bom para scripts, algoritmos lineares e projetos pequenos. POO escala melhor em sistemas grandes porque organiza o código em unidades reutilizáveis e mais fáceis de manter. Na prática, a maioria das linguagens modernas mistura os dois.

# CLASSE 
É um molde, um modelo que descreve como algo deve ser. Ela define quais informações e comportamentos um objeto terá. Sozinha ela não faz nada, só existe para ser usada como base.

# OBJETO
É a coisa real criada a partir de uma classe. Você pode criar vários objetos do mesmo molde, cada um com seus próprios valores. Se a classe é a planta de uma casa, o objeto é a casa construída.

# CONSTRUTOR
É um método especial que roda automaticamente no momento em que um objeto é criado. A função dele é preparar o objeto, definindo os valores iniciais que ele precisa ter pra existir. Sem construtor, o objeto nasceria "vazio", sem nenhuma informação.

#.THIS
É uma palavra que significa "este objeto aqui". Dentro de uma classe, quando você fala this.nome, está dizendo "o nome deste objeto específico que estou usando agora". É o que diferencia um objeto do outro quando vários são criados a partir do mesmo molde.

# ENCAPSULAMENTO
É o princípio de esconder o que é interno e expor só o necessário. Um objeto guarda seus dados protegidos e oferece formas controladas de acessar ou modificar esses dados. É como um caixa eletrônico — você não tem acesso direto ao dinheiro dentro da máquina, só interage com ela pelos botões. Isso evita que alguém mexa nos dados de forma errada ou inesperada.
